StreamVu

Stream Vu helps users find their next movie or show by offering a comprehensive search experience.

Methodology: The primary goal of this project was to gather insights from user research and then build and test a low-fidelity mobile prototype.

Methodology

A problem statement was first developed based on the difficulty of finding certain programs and movies among the plethora of streaming services currently available.The problem statement was then validated through a series of user interviews which were synthesized into various formats to build a compelling case study.

User-Flow

  • This is a step-by-step analysis of how users travel through the app
  • Showing any poetential pain points or issues that can cause friction for users
  • A blueprint used to build the prototype

Storyboard

  • The storyboard is a visual showcase of our users journey from encountering a problem to finding the solution
  • Using the user-persona as the character for the storyboard

Feature-Prioritization

  • A visual of all the basic features for the app
  • Arranged by difficulty of implemantation (low-high) and low-high impact to end users
  • Focus on low complexity features with a high impact

User-Testing Results

  • This shows the results of the user-testing process
  • This feedback was used to itterate and improve the StreamVu prototype

User-Persona

  • This is a more realistic and more detailed approach to the average user
  • Informed by results from user research

Affinity-Diagram

  • This is all the data recieved from user interviews split into different categories
  • This process helps the team to uncover key findings and common pain points among users

Interview results

  • This is a visual representation of all the data recieved during user interviews
  • Gathered from 5 user interviews as well as a user survey
  • Research phase

Proto-Persona

  • This is a snapshot of our ideal user
  • An imagined persona created without input from research
  • Pre-research phase

NSA Redesign

This project was a redesign of the NSAâ€™s government website. Focusing primarily on restructuring the current sites overloaded and confusing navigation.

Methodology: The primary goal of this project was to gather insights from user testing in order to produce a hi-fidelity mockup for both a desktop as well as a mobile browser redesign.

Methodology

An in-depth evaluation of the NSAâ€™s current website was first performed. User tests were also conducted to collect information about existing pain-points. This information was translated into a current site-map as well as a redesigned site-map foucsing on a more enjoable user experience. This sitemap, along with user input was used to create a final high fidelity prototype.

UI Style Tile

  • This style tile informs the team of specific design standards and patterns
  • Includes accepted branding, colors, and iconography as well as text styles and sizing
  • Used to build a polished hi-fi design

Site-Map (Redesigned)

  • This This is the redesigned site-map
  • Less redundancy in navigation options
  • Reduced cognitive load without losing any available information
  • Six main categories reduced to five

Site-Map

  • This site-map shows the entire expanded navbar in an organized format
  • Helps to visualize the sheer size of the current sites unruly navigation
  • Used as a base to redesign a more focused navigation system

Card-Sorting

  • Card-sorting is a method of laying out the entire site navigation in a straightforward all-together format
  • This process helps to inform designers on where to focus for a navigation-centric redesign

Proto-Persona

  • This is a snapshot of our ideal user before any research is conducted
  • The start point used to establish what the user needs/expects from the site
  • Pre-research Phase

PerfectEdge

Perfect Edge is a mobile experience designed to give users the opportunity to aquire high end kitchen equipment on a rental basis.

Methodology: User interviews were conducted and used to establish a problem statement. Which was used to create supporting documentation as well as a functional low and high fidelity prototype.

Methodology

A problem statement was first developed based on the difficulty of finding certain programs and movies among the plethora of streaming services currently available.The problem statement was then validated through a series of user interviews which were synthesized into various formats to build a compelling case study.

User Flow

  • Lays out the userâ€™s movement through the product, mapping out each and every step they take
  • Start with the users first time using the app, all the way through to the end of the checkout process

Storyboard

  • Can help designers understand the flow of peopleâ€™s interaction with a product over time
  • Helps to create empathy for users
  • Simple showcase for people unfamaliar with the UX process

User-Persona

  • Created based on insights gathered from research to represent the different user types that might use the Perfect Edge service
  • Helps to identify with the user and aids in creating a better product

Affinity-Map

  • Used for analyzing and synthesizing user research findings by patterns and themes
  • Summarizes results quickly and effeciently in order to share among team members

User-Interviews

  • These are the results of the interview process categorized by each user interviewed
  • The interview questions are also listed

Proto-Persona

  • Created to give the deisgn team a way to quickly communicate initial ideas of user behavior
  • Based on assumptions, and further checked against actual data obtained through user interviews
